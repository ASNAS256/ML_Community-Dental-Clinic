## Project Overview

This repository is for the collaborative design and implementation of a Health Information Management System (HIMS) for the Community Dental Clinic. The system aims to streamline patient data management, appointment scheduling, and clinical record-keeping.

All contributors are required to **work on their own branches** and submit pull requests regularly for review and integration.

## Project Structure

```
ML_Community-Dental-Clinic/
│
├── app/
│   ├── models/                  # Feature-separated model definitions
│   │   ├── user_model.py
│   │   ├── patient_model.py
│   │   └── ...
│   │
│   ├── <feature>/               # One directory per application feature
│   │   ├── routes/              # All route definitions for the feature
│   │   │   └── routes.py
│   │   ├── templates/           # Feature-specific HTML templates
│   │   └── __init__.py
│   │
│   ├── extensions.py            # All Flask extensions (db, login_manager, migrate, etc.)
│   └── __init__.py              # Application factory
│
├── migrations/                  # Auto-generated by Flask-Migrate
├── venv/                        # Virtual environment (not committed)
├── requirements.txt
├── run.py                       # Application entry point
└── README.md
```

> **Architecture note:** Each feature lives in its own isolated directory under `app/`. Routes are defined inside a `routes/` sub-directory within that feature folder. Models are separated by feature and live in the `app/models/` directory. All Flask extensions (SQLAlchemy, LoginManager, Migrate, etc.) are initialized in `extensions.py` to avoid circular imports.

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/ASNAS256/ML_Community-Dental-Clinic.git
cd ML_Community-Dental-Clinic
```

### 2. Create and activate a virtual environment

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**macOS / Linux:**
```bash
python -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install flask flask_sqlalchemy flask_login flask_migrate
pip install -r requirements.txt
```

> If you encounter missing package errors when running the app, install them individually inside the virtual environment and add them to `requirements.txt`.

### 4. Run the application

```bash
python run.py
```

The app will be available at `http://127.0.0.1:5000` by default.

## Development Guidelines

### Adding a New Feature

1. Create a new directory under `app/` named after the feature (e.g., `app/appointments/`).
2. Inside it, create a `routes/` sub-directory with your route definitions.
3. Define any new models in a dedicated file under `app/models/` (e.g., `app/models/appointment_model.py`).
4. Register any new extensions in `app/extensions.py`.
5. Register the feature's Blueprint in `app/__init__.py`.

### Extensions Pattern

All Flask extensions must be initialized in `extensions.py` and imported from there across the app — **never initialize extensions inside individual feature files**.

```python
# app/extensions.py
from flask_sqlalchemy import SQLAlchemy
from flask_login import LoginManager
from flask_migrate import Migrate

db = SQLAlchemy()
login_manager = LoginManager()
migrate = Migrate()
```

### Running Migrations

```bash
flask db init       # Only needed once
flask db migrate -m "Description of changes"
flask db upgrade
```

## Contributing

1. **Fork** the repository and **create your own branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes following the project structure conventions above.
3. Commit your work with clear, descriptive messages:
   ```bash
   git commit -m "feat: add appointment scheduling routes"
   ```
4. **Push your branch** and open a **Pull Request** for review:
   ```bash
   git push origin feature/your-feature-name
   ```
5. Do **not** push directly to `main`.

> All collaborators are expected to push updates regularly so progress can be tracked and reviewed by the team.
